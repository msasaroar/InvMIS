Project References To (Dependencies) কারণ
InvMIS.Domain — Core entities ও নিয়ম (POCO classes)। অন্য কারও উপর নির্ভরশীল নয়।
InvMIS.Application InvMIS.Domain Services/Interfaces ডোমেইন entity ব্যবহার করে; ব্যবসায়িক নিয়ম প্রয়োগ করে।
InvMIS.Infrastructure InvMIS.Domain, InvMIS.Application EF Core DbContext ও Generic Repository; Application-এর interfaces ইমপ্লিমেন্ট করে।
InvMIS.API InvMIS.Domain, InvMIS.Application, InvMIS.Infrastructure Controllers → Services call; DI কনফিগারেশন; Swagger/JWT/Seeding ইত্যাদি।
Reference Direction:
Domain ⟵ Application ⟵ Infrastructure ⟵ API

ব্যাখ্যা (সংক্ষেপে)

  • InvMIS.API কেবল Controller/Endpoint প্রকাশ করে; এখানে Service গুলো DI এর মাধ্যমে ইনজেক্ট করা হয়।
  • InvMIS.Application এ business logic থাকে; এখানে Interface ও Service থাকে।
  • InvMIS.Infrastructure এ বাস্তব data-access (EF Core, Repository, DbContext) থাকে এবং Application-এর Interface ইমপ্লিমেন্ট করা হয়।
  • InvMIS.Domain এ কেবল Entities/POCO মডেল থাকে—কোনো বাইরের reference নেই (pure core)।
